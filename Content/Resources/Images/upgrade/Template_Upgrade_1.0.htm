About the Gemalto Flare Template

The Gemalto Flare template is a rebranded version of the original SafeNet Flare template. It also adds some features introduced in Flare 10 and 11. Although the template will work in Flare 10, Flare 11 is recommended. The Gemalto Flare template does not support Flare 9.

Changes Introduced in this Version

The changes to the template are minimal, and are primarily focused on Gemalto rebranding. The changes are detailed in the following sections.

Headings

Headings are now all purple (R:108 B:40 G:107) and bold. The autonumber for the h1.chapter style now just inserts a big orange number instead of the text Chapter <n>. These changes are captured in Gemalto_Template.css.

Cover

The cover is all new, with a brown rectangle, purple rectangle, Gemalto logo, and white text. These changes are captured in Cover.htm. The logo is in <project>/Content/Resources/Images/icons/gemalto_logo.png

Variables

The following variables have been updated:

Variable name Previous value New value
CompanyName SafeNet Gemalto
CompanyNameLong SafeNet, Inc. Gemalto NV
Copyright Manually entered text. For example, Copyright <year>.

"Copyright" yyyy

Automatically inserts the word Copyright and the current year. For example, Copyright 2015
DateLong Manually entered text. For example, 15 July 2015

dd MMMM yyyy

Automatically inserts the current day, month, and year. For example, 15 July 2015.
DateShort Manually entered text. For example, July 2015

MMMM yyyy

Automatically inserts the current month and year. For example, July 2015.

DateTime variables were added in Flare 10. You must you Flare 10 or higher to use the new Copyright, DateLong, and DateShort variables.

Cross-Reference Formats

Cross-references now use the {pageref} operator, which creates context-sensitive page references in the PDF targets (above, below, on next/previous page, on page <n>). Cross-references also display differently for the online and print mediums. For print, the referenced paratext is enclosed in quotation marks. For online, there are no quotation marks.

Page Layouts

All page layouts were updated to remove the orange bar from the top of the page. Additional changes are as follows:

FrontMatter.flpgl Updated the Title page to add the the new cover.

Index.flpgl

 Updated to correct a flow issue that caused the Index heading to align with the center of the page.

Master Pages

The template uses a single master page for the HTML output. In the previous version of the template, this file was named SafeNet_Book.flmsp. It has been renamed Default.flmsp in the new template. The content remains the same.

Skin

The skin used for the HTML output has been updated to remove the orange bar and to use the Gemalto logo. Everything else remains the same.

Boilerplate Graphics

All graphics used in the template are located in the <project>/Content/Resources/Images/icons directory.

Deploying the Gemalto Flare Template

The Gemalto template is a drop-in replacement for the SafeNet template. Deploying the template involves moving a few files/directories from the new template into your existing projects, and updating your target files to point to the new files.

To migrate to the template

Open the Gemalto_Flare_Template.flprjzip file in Flare 10 or Flare 11. The project is automatically unzipped to the current directory.

In Windows Explorer, copy the updated files from the template Content/Resources folder to your project:

Go to the template Content/Resources folder.

Copy the Stylesheets/Gemalto_Template.css file to the corresponding folder in your project.

Copy the PageLayouts folder to you project, overwriting the existing PageLayouts folder.

Copy the Images/icons folder to the Images folder in your project

In Windows Explorer, copy the updated files from the template Project folder to your project:

Go to the template Project folder.

Copy the Variables/Default.flvar file to the corresponding folder in your project.

Copy the Skins/Gemalto_HTML5 file to the corresponding folder in your project.

Update your target files to point to the new files:

Open your project in Flare and go to the Content Explorer.

Rename the SafeNet_Book.flmsp in the MasterPages folder to Default.flmsp. Click Update Links when prompted.

Go to the Project Organizer.

Open each target, in turn, and update the General > Master Stylesheet setting to point to the new CSS (Resources/Stylesheets/Gemalto_Template).

For each HTML target, update the following settings:

Skin > Skin Gemalto_HTML5
Advanced > Master Page Resources/MasterPages/Default

Replace the existing Support Contacts table with the updated version. The new table is stored as a snippet in the Content/Resources/Snippets folder.
