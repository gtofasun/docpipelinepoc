Vector Images

Vector images use mathematical formulas to generate visual output such as lines and curves. Unlike bitmap images, vector images can be scaled to any size without loss of quality. To create vector images, SafeNet currently uses Inkscape.

Inkscape is a free, open-source vector graphics drawing tool and editor. Its primary format is Scalable Vector Graphics (SVG), but it can import and export many other formats, such as PNG. Should you choose Inkscape to create your drawings, this section provides important guidelines for using those images in Flare projects.

SVGs can be edited from inside the Flare project by right-clicking the image file in the Content Explorer, and selecting Open with > inkscape.exe.

Prerequisites

Consider the following prerequisites before using SVGs in Flare:

You must install the Java Runtime Environment (JRE) for your workstation (and the server that handles your automatic builds, if applicable). http://www.oracle.com/technetwork/java/javase/downloads/jre8-downloads-2133155.html

Earlier versions of Flare required GhostScript (64-bit version) to render SVGs: https://www.ghostscript.com/download/gsdnld.html. It is not clear whether this is still a prerequisite, but if you encounter issues, this could help.

Colour Fills

SafeNet style vector images are in greyscale. When depicting multiple different objects in an image, use the following fill values:

Icons(Client laptop icon, etc.) 70% Grey
HSMs

30% Grey
Partitions 7.5% Grey

Borders

When creating a shape of any kind, set Stroke style >> Width to 1px. Note that if a shape is scaled via clicking and dragging the border with the Select and Transform Objects tool, this value will also scale; change this value back to 1px before using the image.

To avoid this, resize shapes with the Edit paths by Node tool instead, by clicking and dragging one of the square nodes.

The stroke paint value should be 40% grey, or 969696ff in RGBA, 100% opacity.

Text

Text should be in Arial font to match the template <p> style, and sized at 8pt, 10pt, or 12pt for small, normal, or large text.

Here are some tips for creating text in Inkscape:

Choose your desired text characteristics (font, size, alignment, etc.) in the top ribbon before you click to create your text boxes, especially alignment. Changing the alignment of text later can cause misrendering in Flare. Flare has trouble locating centered text -- it's easier to use left- or right-aligned text and center using spaces.

From the Edit menu, select Preferences > Tools > Text. Under Style of new objects, select Last used style. This will prevent you from having to change the characteristics of each new text box. This setting will persist in subsequent Inkscape sessions.

For gray-shaded text, use a gray color rather than lowering opacity, or text will pixellate when zoomed in.

Flowtext glitch

A text-box feature in Inkscape, described in their manual here, can cause an issue where Flare will import blank SVGs. Create all image text by selecting the text tool, clicking in one spot and entering text, using Enter for line breaks. Clicking-and-dragging to create a text box of a particular size and shape will create a flowtext box which is not supported by the current SVG standard.

If you encounter this issue importing SVGs, look for invisible, empty text boxes that are identified in the Object Properties tab as flowtext. If you find one of these, the easiest way to find the rest is to leave it selected, then use Edit > Select Same > Object Type. You must then delete these boxes, or convert them to regular text with Text > Convert to text.

Image Scaling

There is no filesize benefit in scaling down SVGs. Image size can be dictated by the size of the text (8pt, 10pt, 12pt) and reproduced at its native size, with no need to resize them in Flare's Image Properties. Here are some tips for scaling images in Inkscape:

Scale the Inkscape document to the standard width of 650 pixels or lower by selecting the entire image, then use Object > Transform > Scale and select your desired characteristics.

If possible, add text and borders last, because scaling the entire image will change the point-size of text boxes and the width of borders.

Resize Inkscape documents to the boundaries of your image with Edit > Resize Page to Selection.

Exporting

If you choose to export images to PNG, ensure that the edges of the page fit to the drawing:

Ensure no objects are selected.

From the Edit menu, select Resize Page to Selection.

From the File menu, select Export PNG Image.

When exporting, adhere to the current image size standard: 650px wide or less.

Never delete the source SVG file, store it alongside the exported PNG instead so that you can make edits later.
