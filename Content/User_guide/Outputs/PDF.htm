PDF Outputs

The format of your PDF outputs is defined by the page layouts you apply to the folders and topics in the TOC. You can use the page layouts for books to create a book with a cover, frontmatter, table of contents, chapters, etc., or you can use the Notes page layout to create a technical note consisting of one or more topics. You can use different page layouts to apply Draft or Confidential watermarks to your PDF outputs. See TOCs: Defining the Output Contents, Structure, and Layout for information about applying page layouts to your TOCs.

PDF Books

To create a PDF book for your content, you need to create the content elements required for a book, and structure your TOC as described in PDF TOCs.

PDF Technical Notes

To create a PDF technical note for your content, you need to apply the First page of the Notes page layout to the first topic in the technical note. The fist topic in the technical note must use the format outlined in the following template topic files:

Release notes and generic notes Content/note/noteTemplate.htm
Technical notes (see Note below) Content/technote/TechnoteTemplate.htm

These template files contain the appropriate paragraphs for the note headings and also include proxies for generating the table of contents for the note. If the note is a single topic file, use the mini-TOC proxy. If the note contains multiple topic files, use the output TOC proxy.

Although we provide a release note template and a technical note template, the only difference between the two is the use of the p.subjectNotes style to provide a sub-title for the technical note. In practice, this format is seldom used and has been deprecated. It is recommended that you use the format defined by Content/note/noteTemplate.htm and the Notes page layout.

Building the Example PDFs

You can build any of the example PDFs by building the targets in the Project/Targets/PDF folder. The targets will also tell you which TOC is used to build the file. See Configuring Your Output Targets and TOCs: Defining the Output Contents, Structure, and Layout for more information.
